I am the Home Component

 I am imported at root module level

I need to be imported at root level to have access to md-modules. If we want to take advantage of Lazy Loading, we will need to break out components and only import md-modules that are being used. This will also reduce overall footprint. Note: Presently our Library Module imports 'all' md-modules. Note*: Material design is going to be deprecating the wild card 'MaterialModule' - which automatically pulls in everything material. So, in any event, we will need to refactor at some point.

Input: Floating placeholder spacing fixed

I have put some basic routing in place at the 'root' level. I'll be adding Wijmo as a separate 'Feature Module' and it will be Lazy loaded on demand.

app.routes.ts
import {Routes,RouterModule} from "@angular/router";
import {HomeComponent} from "./home/home.component";
import {BasicComponent} from "./basic/basic.component";
import {SecondaryComponent} from "./secondary/secondary.component";
import {OtherComponent} from "./other/other.component";

const APP_ROUTES = [
{path:'', component: HomeComponent}, // This needs to be imported and declared in app.module.ts in order to have access to md-modules
{path: 'basic', component: BasicComponent} // // This needs to be imported and declared in app.module.ts in order to have access to md-modules
{path: 'secondaryr', component: SecondaryComponent} // // This needs to be imported and declared in app.module.ts in order to have access to md-modules
{path: 'other', component: OtherComponent} // // This needs to be imported and declared in app.module.ts in order to have access to md-modules
{path: 'wijmo', loadChildren: 'app/wijmo/wijmo.module#WijmoModule'} // This is a feature module - notice no import here and no import in app.module.ts.
It has a separate directory and routing file - only imports dependencies it needs. And is loaded on demand. The wijmo routing ts file is imported by the WijmoModule.
